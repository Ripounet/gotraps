It's nice to see that the filtering has detected the impostor. But Here is my greeting was definitely not supposed to be printed.

The problem is that msg, err := greet(name) does declare a new variable msg, and also accidentally shadows the result parameter err. This is tricky because using the short declaration operator := allows mixing new and existing variables, and it works well on a very similar code. It depends whether the scope of the existing variable is exactly the same as the scope of the assignment, or not.
