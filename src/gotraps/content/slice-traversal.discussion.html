range (over a slice) with only one variable yields the index, not the element. To iterate over the elements and ignore the indexes, use the blank identifier : for _, nephew := range nephews. Try it.

The result may surprise a beginner the first time, but Go programmers enjoy the range construct because it gives both the key and the element as iteration variables.

When the type of your slice elements is not int, confusion is unlikely because the compiler won't let you use the index instead of the element: iteration variables are strongly typed and the types would mismatch. But if you're iterating over a []int, you have to carefully name and use the index and the element: the compiler won't shout.
